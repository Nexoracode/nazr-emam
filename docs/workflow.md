# Workflow پروژه نذر امام

این پوشه منبع اصلی هماهنگی پروژه است. قبل از شروع کدنویسی، این سه سند را به‌ترتیب بخوان:

1. [`ONBOARDING-FA.md`](./ONBOARDING-FA.md): معرفی پروژه، معماری، راه‌اندازی، گردش کار روزانه و قوانین کدنویسی.
2. [`API-CONTRACT.md`](./API-CONTRACT.md): قرارداد API بین NestJS و Next.js. هر تغییر API اول اینجا ثبت می‌شود، بعد در کد.
3. [`REVIEW-WITH-CLAUDE-FA.md`](./REVIEW-WITH-CLAUDE-FA.md): روش بازبینی روزانه کد و PRها.

## خلاصه پروژه

«نذر امام» یک سامانه RTL فارسی برای ثبت، پرداخت، پیگیری و مدیریت نذرهاست. کاربر می‌تواند نوع نذر را انتخاب کند، اطلاعات لازم را وارد کند، پرداخت یا رسید را ثبت کند و وضعیت درخواست خود را پیگیری کند. مدیر سامانه هم درخواست‌ها، پرداخت‌ها، گزارش‌ها و پیام‌های کاربران را مدیریت می‌کند.

## معماری فعلی

```text
nazr-emam/
├── apps/
│   ├── web/      -> فرانت اصلی با Next.js
│   ├── api/      -> API و منطق کسب‌وکار با NestJS
│   └── cms/      -> مدیریت محتوای قابل ویرایش
├── packages/
│   └── shared/   -> تایپ‌های مشترک قرارداد API
├── docs/         -> workflow و قراردادها
├── Dockerfile    -> اجرای production هر دو اپ داخل یک container
├── docker-compose.yml
├── turbo.json    -> مدیریت taskهای monorepo
└── package.json  -> workspace ریشه با npm + Turbo
```

## قوانین اصلی

- API روی `http://localhost:3001` اجرا می‌شود.
- Frontend روی `http://localhost:3000` اجرا می‌شود.
- CMS workspace وجود دارد، اما ابزار آن هنوز قطعی نشده است.
- اجرای local با `npm run dev` انجام می‌شود.
- build کل پروژه با `npm run build` انجام می‌شود.
- اجرای production باید با یک Docker container انجام شود.
- هر endpoint قبل از پیاده‌سازی باید در `docs/API-CONTRACT.md` ثبت شود.
- اگر شکل داده بین فرانت و بک تغییر کرد، اول قرارداد و تایپ مشترک به‌روز می‌شود، بعد کد.

## فازهای اجرای پروژه

### فاز ۱: زیرساخت

- تکمیل monorepo با `apps/api` و `apps/web`.
- تنظیم Turbo برای `dev`، `build`، `start` و `lint`.
- اجرای API و frontend داخل یک Docker container.
- افزودن health check و صفحه اتصال اولیه.

### فاز ۲: دامنه نذر

- تعریف نوع نذر، کمپین نذر، درخواست نذر، پرداخت، رسید و وضعیت‌ها.
- طراحی قرارداد API و تایپ‌های مشترک.
- مشخص کردن مدل محتوایی CMS.
- مشخص کردن مسیر کاربر و مدیر.

### فاز ۳: پیاده‌سازی API

- ماژول‌های NestJS برای auth، vows، payments، admin، tickets و notifications.
- validation ورودی‌ها و خطای یکسان.
- مستندات Swagger در صورت نیاز.

### فاز ۴: پیاده‌سازی Frontend

- صفحه اصلی و معرفی نذرها.
- فرم ثبت نذر.
- صفحه پیگیری وضعیت.
- داشبورد کاربر.
- پنل مدیریت.

### فاز ۵: کیفیت و تحویل

- تست build با Turbo.
- تست اجرای Docker.
- review دستی PRها.
- تکمیل README و envها.
