# قوانین CMS — apps/cms

> قوانین ریشه (`/CLAUDE.md`) را هم رعایت کن. این فایل فقط قوانین اختصاصی CMS است.

## مسئولیت این اپ

`apps/cms` برای مدیریت محتوای قابل ویرایش پروژه است:

- متن‌ها و بخش‌های صفحه اصلی
- بنرها و تصاویر
- محتوای معرفی نوع‌های نذر
- پرسش‌های متداول
- صفحات ثابت
- مدیا
- محتوای راهنما و اطلاع‌رسانی

حساب کاربری، پرداخت، درخواست نذر، رسید، گزارش مالی و وضعیت‌های تراکنشی کار CMS نیستند؛ این‌ها متعلق به `apps/api` هستند.

## ابزار CMS

ابزار CMS هنوز قطعی نشده است. تا قبل از تصمیم نهایی:

- Payload، Strapi یا ابزار دیگر نصب نکن.
- schema واقعی یا collection واقعی نساز.
- فقط قرارداد، مدل مفهومی و نیازمندی‌ها را در docs ثبت کن.

وقتی ابزار انتخاب شد، همین فایل با قوانین دقیق همان ابزار کامل می‌شود.

## مرز CMS با API

- CMS فقط محتوای قابل ویرایش را مدیریت می‌کند.
- API فقط داده تراکنشی و امن را مدیریت می‌کند.
- `apps/web` هر دو منبع را مصرف می‌کند:
  - محتوای عمومی از CMS
  - عملیات کاربر و مدیر از NestJS API

## مدل‌های محتوایی پیشنهادی

بعد از انتخاب ابزار CMS، این collectionها احتمالاً لازم می‌شوند:

- `home_page`
- `nazr_type_content`
- `faq`
- `static_pages`
- `media`
- `announcements_content`

نام فیلدها باید با قرارداد `docs/API-CONTRACT.md` و تایپ‌های `@nazr-emam/shared` هماهنگ باشد.

## فارسی و RTL

- محتوای CMS فعلاً فارسی است.
- فیلدهای متنی باید برای نمایش RTL مناسب باشند.
- اگر بعداً چندزبانه شد، اول قرارداد آن در docs ثبت می‌شود.

## چیزهایی که نباید بکنی

- منطق پرداخت یا درخواست نذر را داخل CMS نساز.
- بدون تصمیم کاربر ابزار CMS نصب نکن.
- نام فیلدها را جدا از قرارداد API انتخاب نکن.
- محتوای حساس یا اطلاعات تراکنشی کاربران را در CMS نگه ندار.
